Monitor Management via SCDS API

Get Monitor Classes

The HTTP method to be used is:
URI will be /monitorclass
or /monitorclass/{idorname}
If a Monitor Class ID or Name (partial) is specified, only that class will be returned; otherwise all will be returned.
Parameters: limit, limit
The maximum number of items to return.   offset, offset
Where to start with the results, relative to the beginning.   fields, fields
A comma separated list of field names to be returned, others will be omitted.  
Defaults: 10, 0, All fields






Create a Monitor Class

The HTTP method to be used is:
URI will be /monitorclass
Parameters: name, name
The name you want to give to the monitor class (text).   sql, sql
A SQL statement that is to be used to implement this monitor class.   description, description
A description of the monitor class (text).   type, type
The type of chart e.g. LineChart   delta, delta
Whether the observations are differences.   monitortype, monitortype
The type of monitor class - choose from SQL, CRM, PING, COMMAND.   systemaverage, systemaverage
Whether the system value for a monitor is an average of the values for all the nodes or a sum of the values.   interval, interval
The number of seconds between successive observations. unit unit
The name of the units in which observations are measured (text).

Get Node Monitor Data

The HTTP method to be used is:
URI will be /system/{systemid}/node/{nodeid}/monitor/{monitor}/data
Parameters: start, start
The time to start the first interval to be returned - unix time in seconds. If the start time is specified, then the count (either submitted or defaulted) may be recalculated. The count will never be increased, but it will be reduced if necessary to ensure that the finish time is no more than the value submitted (or defaulted to the present). If no start time is specified, then the start time is computed by taking the finish time and going back by the specified (or defaulted) count of intervals.   finish, finish
The time to stop the last interval to be returned - unix time in seconds. Default is the present time. If a start time is specified, then the finish time may be adjusted so as to have an integral number of intervals from the start time up to the finish time.   interval, interval
Number of seconds in each time interval. The number of intervals may be specified below, although it may be overridden if a start time is given - see the tip for the start field. The given interval is always used and never recalculated. There is a default value in the API configuration file.   count count
The number of time intervals for which interpolated observations are wanted. It may be overridden by a calculation based on start time - see the tip for start. Otherwise, the start time is calculated by taking the finish time and going back by the specified number of time intervals. There is a default value in the API configuration file.










Get Raw Node Monitor Data

The HTTP method to be used is:
URI will be /system/{systemid}/node/{nodeid}/monitor/{monitor}/rawdata
Parameters: start, start
The time to start selecting observations to be returned - unix time in seconds. If the start time is specified, it will be used. If no start time is specified, then the start time is computed by taking the finish time and going back by the specified count of intervals. The start time is the first variable in a SQL BETWEEN clause.   finish, finish
The time to stop selecting observations to be returned - unix time in seconds. Default is the present time. The stop time is the second variable in a SQL BETWEEN clause.   interval, interval
Number of seconds in each time interval. Only used to compute the start time if none is specified.   count count
Only used to compute the start time if none is specified.
